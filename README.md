# 🌤 朝の天気&災害情報をLINEに届けるシステム(石狩市版)

## これは何?

毎日自動で、あなたのLINEにこんなメッセージが届くようになります。

- **朝6:30** … 石狩市周辺の今日の天気・降水確率・気温(警報が出ていればそれも)
- **昼12:00** … 大雨警報などの災害情報が出ている場合だけ、お知らせ

パソコンをつけっぱなしにする必要はありません。**完全無料**です。

## 仕組みをざっくり説明

登場するのは3つのサービスです。全部無料で使えます。

1. **気象庁** … 天気と警報のデータを公開している国の機関。ここからデータをもらいます。
2. **LINE** … 「自分専用のお知らせBot(ロボットアカウント)」を作り、そこからメッセージを受け取ります。
3. **GitHub(ギットハブ)** … プログラムを預けておくと、毎日決まった時間に代わりに実行してくれるサービス。

つまり「GitHubが毎朝プログラムを動かす → プログラムが気象庁から天気をもらう → LINEのBotがあなたに送る」という流れです。

# セットアップ手順(所要時間:約30分)

## ステップ1:LINEのBot(お知らせ用アカウント)を作る

1. スマホかパソコンのブラウザで **LINE Developers**(https://developers.line.biz/ja/)を開く
2. 「コンソールにログイン」を押し、**普段使っているLINEアカウント**でログイン
3. 初回は開発者登録画面が出るので、名前とメールを入れて登録(無料)
4. 「プロバイダー」を作成する画面が出たら、名前は何でもOK(例:「じぶん用」)
5. 「Messaging API」を選んでチャネル(=Bot)を作成
   - チャネル名:例「天気お知らせ」
   - 業種などは近いものを適当に選んでOK
6. 作成できたら、画面上部のタブから **「Messaging API設定」** を開く
7. ページ下の方にある **「チャネルアクセストークン(長期)」の「発行」ボタン**を押す
   - 出てきた長い文字列をメモ帳などにコピーしておく(あとで使います)⭐️トークン　2010675174　　ac9a6a586af275ed3d6565a1c67d402f
8. 同じページの上の方にある **QRコード** を自分のスマホのLINEで読み取り、このBotを**友だち追加**する
9. タブを **「チャネル基本設定」** に切り替え、一番下にある **「あなたのユーザーID」**(Uから始まる文字列)をコピーしてメモ(あとで使います)⭐️ユーザーID

> 💡 Botから「応答メッセージ」が自動で返ってくるのが気になる場合は、Messaging API設定内の「応答メッセージ」を無効にできます(必須ではありません)。

## ステップ2:GitHubのアカウントを作る

1. https://github.com/ を開き、「Sign up」からアカウントを作成(無料)
   - メールアドレスとパスワードを決めるだけです
2. ログインした状態で次のステップへ

## ステップ3:プログラムをGitHubに置く

1. GitHubの画面右上の「+」→ **「New repository」** を押す
2. Repository name に好きな名前を入力(例:`weather-line`)
3. **「Private」を選ぶ**(自分だけが見られる設定。おすすめ)
4. 「Create repository」を押す
5. できたページの中にある **「uploading an existing file」** というリンクを押す
6. ダウンロードしたこのzipを解凍し、中のファイルを**フォルダごとドラッグ&ドロップ**でアップロード
   - `weather_notify.py`
   - `README.md`
   - `.github` フォルダ(この中に予約表が入っています)
   - ⚠️ `.github` フォルダはパソコンの設定によっては見えないことがあります。見えない場合は「隠しファイルを表示」をオンにしてください(Windows:エクスプローラーの「表示」→「隠しファイル」/ Mac:Finderで Command+Shift+.)
7. 下の「Commit changes」ボタンを押して保存

> 💡 ドラッグ&ドロップで `.github` フォルダがうまく上がらない場合:「Add file」→「Create new file」でファイル名に `.github/workflows/morning-weather.yml` と入力し、ymlファイルの中身をコピペする方法でもOKです。

## ステップ4:LINEの「合言葉」をGitHubに登録する

ステップ1でメモした2つの情報を、GitHubに安全に預けます。

1. リポジトリのページ上部の **「Settings」** タブを押す
2. 左メニューの **「Secrets and variables」→「Actions」** を押す
3. 緑の **「New repository secret」** ボタンを押して、次の2つを1つずつ登録:

   | Name(そのまま入力) | Secret(貼り付ける内容) |
   |---|---|
   | `LINE_CHANNEL_ACCESS_TOKEN` | ⭐️トークン(ステップ1-7の長い文字列) |
   | `LINE_USER_ID` | ⭐️ユーザーID(ステップ1-9のUから始まる文字列) |

> 💡 ここに入れた情報は暗号化されて、他人からは見えません。

## ステップ5:テストしてみる

1. リポジトリ上部の **「Actions」** タブを押す
2. 初回は「ワークフローを有効にしますか?」と聞かれるので有効にする
3. 左の「天気と災害情報をLINEに通知」を押す
4. 右側の **「Run workflow」** ボタン → 緑の「Run workflow」を押す
5. 1〜2分待って、自分のLINEに天気のメッセージが届けば **成功です!🎉**

あとは何もしなくても、毎朝6:30と昼12:00に自動で動き続けます。

# よくある質問

**Q. お金はかかりますか?**
かかりません。気象庁のデータは無料、GitHubの自動実行も無料枠内、LINEも無料プラン(月200通まで送信可)で足ります。毎日2通でも月62通程度です。

**Q. 通知の時間を変えたい**
`.github/workflows/morning-weather.yml` の `cron` の数字を変えます。書き方は「分 時」の順で、**日本時間から9時間引いた時刻**にします。
例:朝7:00にしたい → 7時−9時間=前日22時 → `"0 22 * * *"`

**Q. 昼にも毎日必ず通知がほしい**
今の設定では、昼は「警報・注意報が出ているときだけ」通知します(平和な日は静かです)。毎日欲しい場合は言ってもらえれば変更できます。

**Q. 通知が数分遅れる**
GitHubの無料の自動実行は、混雑時に数分〜十数分遅れることがあります。仕様なのでご了承ください。

**Q. どんな災害情報が届くの?**
気象庁が石狩市に発表する警報・注意報すべてです。大雨・洪水・暴風・大雪・雷・なだれなど。緊急度が分かるように 🚨特別警報 / ⚠️警報 / 🟡注意報 のマークを付けています。
